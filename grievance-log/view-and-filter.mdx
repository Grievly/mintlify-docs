title: "View and filter grievances"
description: "Find grievances by facility, status, category, assignee, date, and search terms."
---

Use the grievance list to see open work, find older records, review complaint trends, and prepare for leadership or survey review.

For sidebar queue shortcuts, see [Grievance queues](/grievance-log/grievance-queues).

For top bar search, recently viewed grievances, result previews, and search-side filters, see [Navigate and search](/getting-started/navigate-and-search).

## Filter the list

Use filters to narrow the grievance list by:

- Facility
- Status: Pending Approval, Overdue, or High Priority
- Category
- F-tag
- Assignee
- Date range
- Other filters, such as Has attachments, Public submission, Resident Council, employee involved, third party concern, or anonymous reporter

Open grievances are shown by default. Use **Show resolved grievances** when you need resolved records.

Use **Reset filters** to clear the current table filters while keeping the same queue view.

## Search grievances

Use search when you know part of the resident name, title, grievance ID, category, concern type, complaint description, or other visible case detail.

The list search is tied to the current table view. For broader app-wide searching, use the top bar search.

## Open a grievance

Select a grievance row to view its full case record, including details, follow-up, timeline entries, attachments, and resolution information.

Rows can include title-prefix indicators for public submissions, attachments, High Priority, and overdue work.

## Sort the list

The grievance list defaults to **Created** descending order.

The Created column shows that default sort state even when the URL does not include sort parameters.

You can also sort by received date, SLA, and grievance ID.

## Export or print from the list

The grievance list can generate reports, exports, and print summaries from the current filtered view or from selected rows.

When you generate from the current view, Grievly includes all matching rows, not just the visible page.

When you generate from selected rows, only those selected grievances are included.

Selecting one or more rows opens row actions for **Export** and **Print summary**.

## Keep work moving

Use the list to check which grievances still need ownership, follow-up, documentation, written decisions, or final resolution. Filtering by assignee and status is the fastest way to find stalled work.
