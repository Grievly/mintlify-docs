Follow-up is now documented through the grievance case timeline.

For the current workflow, see [Work a grievance case](/grievance-log/work-a-grievance-case), [Mentions and notifications](/grievance-log/mentions-and-notifications), and [Attachments](/grievance-log/attachments).

Use comments to record investigation notes, actions taken, and coordination. Use attachments for supporting files.

## Add a note

1. Open the grievance.
2. Use the comment composer below the timeline.
3. Enter the comment.
4. Save the entry.

## Mention teammates

Use `@` in a follow-up note when a teammate needs to review or take part in the case. Select the person from the mention list before saving.

Mentioned teammates receive a notification and are added to the grievance's notification audience so they can follow future activity.

## Reference another grievance

Use `#` in a follow-up note to link another grievance, such as a related concern, repeat complaint, or earlier report from the same situation.

When you reference another grievance, Grievly links to that record and adds a timeline mention on the referenced grievance so reviewers can see how the cases are connected.

For more detail, see [Mentions and notifications](/grievance-log/mentions-and-notifications).

## Add actions taken

Use comments to document meaningful steps such as:

- Speaking with a resident or representative
- Reviewing a chart or internal record
- Interviewing staff
- Escalating to leadership
- Assigning corrective action
- Completing a follow-up task

## Write notes for review

Good follow-up notes are specific, dated, and tied to the issue being reviewed. Avoid vague updates like "handled" when the record needs to show what was reviewed, who was contacted, and what changed.

## Attach files

Use attachments for supporting documents that belong with the case record.

Do not upload files unrelated to the grievance.
