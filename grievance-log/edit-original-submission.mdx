---
title: "Edit an original submission"
description: "Correct intake details from the original submission dialog while preserving the case timeline."
---

The **Original submission** dialog shows the source intake packet for a grievance. It opens in read-only mode so staff can review what was captured before making changes.

Use the editor when the original intake record has a typo, missing field, or incorrect detail that should be corrected in the case record.

## What you can edit

Open grievances can be edited from the original submission dialog.

Depending on how the grievance was created, editable fields can include:

- Facility, for internally created grievances.
- Resident name, room, and unit.
- Occurrence date and shift.
- Employee involvement and involved employee names.
- Outside-party involvement.
- Concern details.
- Anonymous-report flag.
- Reporter name, relationship, email, and phone.

Some fields remain read-only in the original submission dialog. For example, public submitter type is preserved as source context, and the internal assignee is edited from the case metadata rail.

## Edit the submission

1. Open the grievance.
2. In the timeline opening item, select **View original submission**.
3. Select the pencil icon in the top-right of the dialog.
4. Update the fields that need correction.
5. Review the **Unsaved changes** indicator at the bottom of the dialog.
6. Select **Save changes**.

If no fields have changed, the save action stays disabled.

## Public submissions

Public intake records keep their source context. The dialog shows whether the submission came from the public intake form and which submitter type was selected.

For public submissions:

- Employee and family-member submissions show reporter contact fields.
- Resident submissions show the resident as the person filing the complaint.
- Anonymous submissions keep reporter identity hidden unless the record is intentionally updated.
- The concern text and occurrence date can be corrected when the grievance is still open.

If the public submitter type was **resident**, changing the resident's first or last name also updates the reporter name used for that source record.

## Internal submissions

Internal submissions can include more operational intake details.

Use the dialog to correct:

- Facility selection.
- Resident details.
- Date, shift, or Resident Council meeting flag.
- Employee or outside-party involvement.
- The concern narrative.
- Reporter details.

Use the case metadata rail for assignment, category, and F-tag changes. Use the header action menu to edit the title or print a case summary.

## Locked records

The original submission editor is unavailable when the grievance is locked.

A grievance is locked when it is resolved or pending resolution approval. This preserves the review record while sign-off or final documentation is in progress.

If new information arrives after resolution, an administrator can reopen the grievance and continue work from the timeline.

## What happens after saving

Grievly updates the case record and writes activity events for the changed fields, such as resident details, concern text, occurrence date, or anonymous-report changes.

The edit does not remove comments, attachments, resolution notes, or prior timeline activity.
