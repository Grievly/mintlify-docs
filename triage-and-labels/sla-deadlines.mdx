---
title: "SLA deadlines"
description: "Understand due dates, overdue work, and facility-level grievance SLA settings."
---

Each grievance can have a due date used to track timeliness.

Grievly uses the due date to calculate SLA progress and identify overdue work.

## At a glance

| Step | What happens |
| --- | --- |
| Received time | Grievly uses the grievance received time |
| Due date | The selected facility's default SLA sets the due date window |
| Overdue | Open grievances past the due date appear as overdue |
| Resolved | Resolved grievances no longer appear as overdue |

## Facility default SLA

New grievances use the selected facility's default grievance SLA length.

Administrators can change this from **Facilities > [Facility] > Configuration > Grievance SLA**.

The default is measured in hours from the received time.

Each facility can use a different default SLA.

## Overdue

A grievance is overdue when:

- It is still open.
- Its due date is in the past.

Overdue is derived from the due date. It is not a separate saved status.

Resolved grievances are no longer overdue.

## Where SLA appears

SLA information can appear in:

- Dashboard attention sections.
- The Overdue sidebar queue.
- The grievance list **Due** column.
- The grievance detail status and case metadata rail.
- Analytics, including SLA breach metrics.

| Surface | How to use it |
| --- | --- |
| Dashboard | Spot due-soon and overdue work |
| Overdue queue | Open the list of active past-due grievances |
| Grievance list | Sort or scan by due date and SLA state |
| Case metadata | Review the exact due date while working the case |
| Analytics | Review timeliness trends over time |

## Reopened grievances

When a resolved grievance is reopened, the user chooses a reason and Grievly preserves the original due date and SLA history. Reopening does not grant a new SLA window.

## Good practice

Use SLA as an operational reminder. It helps teams spot stalled work, but it does not replace the facility's judgment about urgency, required follow-up, or documentation quality.
